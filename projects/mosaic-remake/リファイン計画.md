# 「モザイクかけ太郎」リファイン計画

## 1. 概要
現状の「モザイクかけ太郎」ソースコード(`app_v18.py`)および仕様(`りどみ.txt`)を評価した結果に基づき、ユーザビリティの向上（操作負担の最小化、直感的UI、柔軟性）と保守性向上を目的としたリファイン（再設計・再実装）計画を立案しました。

## 2. 現状の評価と課題
- **UI/ユーザビリティ**: 
  - TkinterベースのためUIの表現力が限られており、モダンさに欠ける部分があります。
  - D&D機能はありますが、パラメータ画面に設定項目が多用されており、初心者には「推論サイズ」等の専門用語が敷居を高くしている可能性があります。
  - AIの誤検知に対する柔軟な手動修正（キャンバス上で特定の誤検知枠をクリックして消す、手動で枠を追加するなど）機能がなく、検知感度の全体調整に頼るしかないため柔軟性が不足しています。
- **保守性と拡張性**: 
  - `app_v18.py` が約1100行の単一ファイルとなっており、画像処理（OpenCV等）、AI推論（YOLO）、GUI構築、イベント処理が密結合しています。今後のモデル追加や細かい機能改修が困難になりやすい構造です。
- **日本語環境対応**:
  - `cv2.imread/imwrite` を使用した処理が見受けられ、Windows環境で日本語を含むパスを処理する際にエッジケースでエラーや文字化けが発生するリスクがあります。

## 3. リファインの目標
設計思想である「操作負担の最小化、直感的UI、柔軟性」を実現するため、以下の目標を設定します。
また、将来的な**商用利用**を見据え、堅牢性や保守性、ライセンス面に配慮した設計を行います。

1. **直感的でモダンなUIへの刷新 (PySide6の活用)**: 
   - 画像プレビュー上でのスムーズなズーム・パンと、手動での直感的なマスク修正（誤検知のクリック削除機能など）を実装し、柔軟性を飛躍的に高めます。
   - ※ GUIフレームワークとして[PySide6(LGPLライセンス)](https://www.qt.io/pricing)を採用することで、商用利用時のライセンス制約を回避しやすいようにします。（PyQt6はGPLのため基本避けます）
2. **操作負担の最小化**: 
   - 「シンプルモード」の実装で、普段使いのユーザーが迷わず一括処理ができるようにUIをスッキリさせます。
3. **モジュール分割による保守性向上と堅牢な例外処理**: 
   - ロジック部分（推論、画像処理など）とUI部分を明確にディレクトリ分離し、疎結合なアーキテクチャに移行します。
   - 処理中の予期せぬエラーでアプリがクラッシュしないよう、商用グレードの例外処理（UIへの親切なエラー通知・リトライ等）を組み込みます。
4. **完全な日本語パス対応**: 
   - 文字化けや読み込みエラーを完全に防ぐセーフティなファイル処理（`cv2.imdecode`等の利用）を徹底します。
5. **起動・配布手段の提供**: 
   - ダブルクリックで簡単に起動できるバッチファイル（`通常起動.bat`等）を実装し、将来的なPyInstaller等を使った単一実行ファイル(.exe)化も容易な構造とします。

## 4. 提案するモジュール構造 (AutoMosaic)
`E:\自作アプリ集\モザイクかけ太郎\AutoMosaic` 内で再構築を行う想定です。

```text
AutoMosaic/
│
├── core/                  # コアロジック (AI推論, 画像処理)
│   ├── processor.py       # YOLO推論管理モジュール
│   ├── filter_utils.py    # モザイク・ぼかし等の加工処理ロジック
│   └── config.py          # 設定データのロード・セーブ管理
│
├── ui/                    # ユーザーインターフェース
│   ├── main_window.py     # メインウィンドウ
│   ├── preview_canvas.py  # プレビュー、手動マスク修正、ズーム/パン機能
│   └── param_panel.py     # パラメータ設定UI (シンプル/詳細モード対応)
│
├── utils/                 # ユーティリティ
│   └── file_io.py         # 日本語パスセーフなファイル入出力関数
│
├── main.py                # アプリケーション起動用エントリーポイント
└── 起動.bat              # ダブルクリック起動用バッチファイル
```

## 5. 実装ステップ
- **フェーズ1：基盤・コアロジックの分離と整備**
  - 既存コードから `MosaicProcessor` および `ImageFilterUtils` を抽出し、`core` ディレクトリにモジュールとして分離。日本語パスに強いファイルI/Oを `utils` に実装。
- **フェーズ2：新UIの構築**
  - PySide6（またはモダン化に適したGUIフレームワーク）を用いて、メインウィンドウおよびパラメータパネルを構築。D&D機能の統合。
- **フェーズ3：インタラクティブプレビューの実装（目玉機能）**
  - 直感的UI・柔軟性の要として、プレビューキャンバス上で検出結果を直接確認・編集（個別削除・追加）できる仕組みを実装。
- **フェーズ4：テスト・調整とリリース準備**
  - 日本語ファイル名を含むデータでの動作検証、動作確認用バッチファイルの作成。
