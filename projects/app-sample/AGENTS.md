# app-sample Rules

## この案件の位置づけ

- このファイルは `app-sample` 固有の前提だけを扱う。
- Workspace共通ルールは上位の `AGENTS.md` を優先する。
- 実装手順や共通チェックは `docs` と `.agents/skills` を参照する。

## 使用ライブラリと構成

- 標準構成は React、TypeScript、Vite、Vitest、Testing Library を前提にする。
- `src` は `components`、`features`、`services`、`state` に責務分離して扱う。
- `tests` では主操作、危険操作、エラー表示の回帰を優先して確認する。

## UI前提

- 主操作は「保存」と「プレビュー」を中心にし、補助操作は控えめに置く。
- 入力フォームは設定、内容、実行結果で分け、初心者が読む順に並べる。
- 保存前に迷いやすい入力項目は近くにまとめる。

## 危険操作

- 削除、上書き、外部送信、初期化は確認ダイアログを必須にする。
- 確認文には対象名、影響、取消不能かどうかを含める。
- 危険操作ボタンは通常操作と色と文言を分ける。

## エラー表示

- エラーは画面上で消えずに確認できる場所へ表示する。
- 項目単位の入力エラーは該当項目の近くに出す。
- 通信や保存の失敗では、原因候補と次の行動を短く示す。

## この案件で確認を優先するもの

- 保存とプレビューの到達性
- 危険操作の確認導線
- 入力エラーと保存失敗時の再試行導線

## build / run / test

- 開発起動: `npm run dev`
- テスト: `npm run test`
- 必要に応じた確認: 保存、プレビュー、危険操作、エラー表示の回帰

## 変更時の必須セット作業

- 更新する docs: `AGENTS.md`、`START_HERE.md`、`PROJECT_SUMMARY.md`
- 最低限の確認コマンド: `npm run test`
- 最低限の手動確認: 保存、プレビュー、危険操作の確認導線、入力エラー表示
- 失敗時に先に見るログやファイル: ブラウザコンソール、開発サーバーログ、対象 feature の state / service 実装
