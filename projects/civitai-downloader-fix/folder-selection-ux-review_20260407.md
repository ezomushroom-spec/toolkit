# 保存先指定 UX 考察 2026-04-07

## 目的

- ダウンロードキューにおける保存先指定の操作性を、利用者視点と実装/運用視点の両方から整理する
- 現行構成を大きく壊さずに改善余地を明確にする

## 現行導線

### 1. 保存先の実体は「フォルダプリセット」

- 設定タブでプリセット名とパスを管理する
- ダウンロードタブでは各ジョブの「保存先」列にプリセット名だけが表示される
- 実パスはキュー一覧では見えない

### 2. プリセット作成は設定タブ依存

- `+ 新規` で空のプリセットを作る
- 名前はセル直接編集、パスは `パス列ダブルクリック` でフォルダ選択する
- 最後に `設定を保存` を押して確定する

### 3. ジョブ側の変更はコンボボックスのみ

- `pending` / `pending_unresolved` のジョブだけ保存先変更できる
- `queued` 以降はテキスト表示に変わる
- 一括変更は右クリックメニュー経由

## 操作性の主な問題

### 1. 保存先の実パスが見えず、どこに保存されるか即判断しづらい

- ダウンロードタブではプリセット名しか見えない
- `（Unsorted）` も実際のパスが一覧からは分からない
- 初心者は「名前を選んだが、保存先の実体がどこか」を確認しにくい

### 2. プリセット作成とジョブ設定が別タブに分かれ、思考が往復する

- 保存先を作る場所は設定タブ
- 保存先を割り当てる場所はダウンロードタブ
- ジョブ投入後に保存先を整える作業が分断され、流れが止まりやすい

### 3. パス指定が `ダブルクリック` 依存で発見しづらい

- 明示的な `参照` ボタンがない
- `新規プリセット` を押した直後に何をすべきかが見えにくい
- ダブルクリック操作を知らないと、空プリセットのまま止まりやすい

### 4. 保存先変更可能なタイミングが分かりづらい

- `pending` 中だけ編集可、開始後は固定になる
- ただし UI 上で「いつまで変えられるか」の説明がない
- 開始後に誤りに気づくと、利用者はキャンセルや削除を選びがち

### 5. 一括変更導線が右クリック依存で見つけにくい

- 複数選択後の主要操作として前面に出ていない
- 「チェックしたら上部に一括保存先変更が出る」構造ではないため、初心者には発見が難しい

### 6. 保存成功の手応えが弱い

- 設定タブでプリセット編集後、反映の見え方が弱い
- ダウンロードタブでコンボ変更した際も、実パス確認や軽いフィードバックがない

## 改善の方向

## A. 最小変更で効く改善

- ダウンロードタブの保存先列ツールチップに実パスを出す
- `（Unsorted）` にも実パスツールチップを出す
- 保存先列ヘッダか上部に「開始前のみ変更可」の説明を出す
- 右クリックだけでなく、上部に `保存先を一括変更` ボタンを追加する
- 設定タブのパス列に `ダブルクリックで選択` の補助文を出す

## B. 中規模だが効果の高い改善

- 設定タブのプリセット表に `参照` ボタン列を追加する
- `新規プリセット` 作成直後にフォルダ選択をそのまま開く
- ダウンロードタブに `新規保存先を作成` のショートカットを置き、設定タブへ移動または簡易ダイアログを開く
- ジョブ行の保存先コンボで、項目のツールチップに実パスを出す

## C. 構成を保ったまま理想に寄せる案

- ダウンロードタブ上部に `保存先管理` セクションを追加する
- ここで `現在選択ジョブの保存先`, `新規プリセット`, `一括変更` をまとめる
- 設定タブは詳細管理、ダウンロードタブは運用時の即変更という役割分担にする

## 推奨順位

1. 実パス可視化
2. パス指定操作の明示化
3. 一括変更導線の表面化
4. ダウンロードタブからの簡易プリセット追加

## 今回の結論

- 一番の詰まりは「保存先の実体が見えない」こと
- 二番目は「設定タブで作ってから戻る」分断
- 現行構成を大きく壊さずに改善するなら、まずは `実パス表示` と `参照ボタン化` と `一括変更の前面化` が効果的
