# 保存先指定の操作性考察 2026-04-07

## 対象

- `E:/civitai_downloader/ui/download_tab.py`
- `E:/civitai_downloader/ui/settings_tab.py`

## 目的

- ダウンロードキューでの保存先指定について、初心者が「どこに保存されるのか」「どう変えるのか」で迷う点を整理する

## 現状の導線

1. 設定タブでフォルダプリセットを作る
2. プリセット名やパスをテーブル編集する
3. `設定を保存` を押す
4. ダウンロードタブで各ジョブの `保存先` 列からプリセット名を選ぶ

## 主要な迷いポイント

### 1. 保存先の実体がキュー画面で見えない

- ダウンロードタブではプリセット名しか見えない
- 実際のフォルダパスは設定タブへ行かないと分からない
- `（Unsorted）` も名前だけで、実パスの確認導線がない

### 2. 保存先を新規作成する場所が分かりにくい

- キュー画面では既存プリセットを選べるだけで、新規作成はできない
- 初回利用者は「候補にない保存先をどこで増やすか」が分かりにくい

### 3. 設定タブの操作が発見しづらい

- 新規プリセット作成後、名前やパスはテーブル直接編集
- パス選択は `パス列ダブルクリック` 依存で、明示的な `参照` 操作がない
- 保存も `設定を保存` まで確定しない

### 4. 保存タイミングのルールが画面ごとに違う

- ダウンロードタブの保存先変更は即時反映
- 設定タブのプリセット編集は保存ボタン押下まで未確定
- 利用者は「いまの変更が保存済みか」を直感しづらい

### 5. 編集可能条件が分かりにくい

- 保存先変更は `PENDING` / `PENDING_UNRESOLVED` のみ
- 開始後はテキスト表示に変わるが、なぜ変えられないかの説明がない

## 採用候補

### A. 最小改善

- ダウンロードタブで保存先セルのツールチップに実フォルダパスを出す
- `（Unsorted）` にも実パスのツールチップを出す
- 保存先列の近くに「候補がない場合は設定タブで追加」と分かる補助文を入れる

### B. 中程度改善

- ダウンロードタブの保存先コンボ末尾に `保存先を管理...` を置いて設定タブへ遷移できるようにする
- 設定タブのプリセット表に `参照` ボタンまたは明示的なフォルダ選択操作を追加する
- 保存後に「プリセットを保存しました」を見える形で出す

### C. 大きめ改善

- ダウンロードタブ内で一時的に新規保存先作成まで完結できるようにする
- ただし既存構成と責務が増えるため、第1候補にはしない

## 推奨

- 現段階では `A + B の一部` が妥当
- 理由:
  - 既存構成を壊さず、迷いの大半を減らせる
  - 保存先管理の正本は設定タブに残したまま、キュー画面からの発見性だけ上げられる
  - いきなりキュー画面へ作成機能を増やすより安全

## 優先順

1. 実フォルダパスの可視化
2. 設定タブへの遷移導線
3. 設定タブのパス選択操作の明示化
4. 編集可能条件の説明

## 未確認

- 実画面でラベルや余白を足したときの圧迫感
