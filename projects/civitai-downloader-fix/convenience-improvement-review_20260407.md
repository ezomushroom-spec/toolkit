# 利便性改善レビュー 2026-04-07

## 目的

- `civitai_downloader` を日常利用しやすくするために、修正すべき仕様、UI、追加機能を優先度付きで整理する
- 現行構成を大きく壊さず、段階的に改善できる内容へ落とす

## 観点

- 利用者視点
- 運用保守視点
- 実装現実性

## 結論

- 便利さの主論点は、`投入から開始までの流れの分かりやすさ`、`失敗時の復旧しやすさ`、`完了後の扱いやすさ` に集約される
- 先に直すべきなのは見た目の装飾ではなく、`状態の見える化` と `次に何をすればよいかの提示` である
- 機能追加は必要だが、まずは既存導線の詰まりを減らす小改修の効果が大きい

## High

### 1. ジョブの失敗理由と復旧操作を一覧上で完結させる

- 現状でも `failed` とツールチップはあるが、一覧だけでは次の行動が弱い
- `リトライ`, `URLを開く`, `エラー詳細`, `ログを見る` を失敗行の近くへ寄せたい
- 右クリック依存ではなく、失敗行の状態セルか詳細ペインで復旧導線を見せるべき

### 2. キュー上部に「次にやる主操作」を集約する

- `選択を開始`, `全て開始`, `保存先を一括変更` はあるが、現在の選択状態に応じた案内がない
- 例えば `3件選択中 / 開始可能2件 / 未解決1件` のような要約と、対応する主操作を並べると迷いが減る
- `開始できない理由` を一覧とは別に上部でも示したい

### 3. ダウンロード中の安心感を上げる状態表示が必要

- `DL中` と進捗バーだけでは、止まっているのか待機中なのか判別しにくい
- `通信中`, `レジューム中`, `移動中`, `キャンセル中` の段階が見えると体感が大きく改善する
- 長時間ジョブでは速度、残り時間、最終更新時刻のどれかが欲しい

### 4. 初回設定の成功条件を明示する

- 現在は API キー、保存先プリセット、ダウンロード開始までの必須理解が分散している
- 初回起動時または設定タブ上部に `最初にやること` を短く出すべき
- 最低でも `APIキーは任意か必須か`, `保存先未設定時はどこに行くか`, `開始は手動であること` は明示したい

### 5. BatchDialog の重複排除と選択件数の追従

- 同一モデルの重複投入は、便利さよりノイズと事故を増やす
- 手動チェック変更で件数表示が追従しない点も操作不信につながる
- 一括追加は大量投入導線なので、ここは UX 上の優先度が高い

## Medium

### 6. ダウンロード行の詳細ペインまたは下部インスペクタを追加する

- 現状は一覧に情報を詰めており、エラー詳細、保存先実体、URL、モデル情報が散っている
- 行選択時に下部へ `保存先 / page_url / download_url / model_id / error_message / dest_path` を出せると復旧しやすい
- 一覧の密度を保ちながら情報量を増やせる

### 7. 通知タブに実行中フィードバックを付ける

- `手動チェック実行` は押下後の体感が弱い
- ボタン無効化、進行中表示、件数進捗、最終チェック時刻の表示がほしい
- 通知が0件でも「チェック済み」であることが分かる表示が必要

### 8. 設定保存の反映タイミングを統一して見せる

- 設定タブは `保存` で確定、キュー側の保存先変更は即時反映で、タイミングが混在している
- 利用者には `即時反映` と `保存ボタンで反映` の違いが見えにくい
- 各操作に `今すぐ反映` / `保存時に反映` の補助文を付けるだけでも理解しやすくなる

### 9. 完了ジョブの後処理導線を強くする

- 現状は右クリックの `保存先を開く` に寄っている
- 完了件数が増えると、`最新完了を開く`, `完了だけ絞り込み`, `保存先を開く` の表導線がある方が便利
- 完了直後のトーストから保存先を開けてもよい

### 10. フィルタと並び替えを追加する

- ジョブが増えると `failed` や `pending_unresolved` を探しにくい
- 最低でも `全件 / 実行中 / 失敗 / 完了 / 未解決` フィルタがほしい
- 並び替えは `新しい順 / 状態順 / 保存先順` があると運用しやすい

### 11. 保存先プリセットを運用タブから補助追加できるようにする

- 既に実パス可視化と一括変更導線は改善済みだが、保存先の新規作成はまだ設定タブ依存
- ダウンロードタブから `新しい保存先を作る` へ短絡できるだけで往復負荷が減る

## Low

### 12. キュー投入時の自動補完を増やす

- ページ URL から追加した時点で、推奨保存先やモデル種別がより早く見えると気持ちよい
- ただし API 呼び出しや待ち時間との兼ね合いがあるため優先度は中以下

### 13. ブラウズタブの収集補助を増やす

- 現状でも右クリックと一括追加はある
- さらに `このページのモデルを収集` の結果をタブ側に軽く要約できると便利
- ただしキュー側の詰まり改善より先ではない

### 14. ログビューアや診断パネル

- 保守には有効だが、一般利用者には常時見せる必要は薄い
- `トラブル時だけ開く` 形なら有効

## 実装規模で分けた推奨順

### 小

- BatchDialog の重複排除
- BatchDialog の選択件数追従
- 通知タブの実行中表示
- キュー上部に選択件数と開始可能件数の表示
- 完了トーストに `保存先を開く` 導線
- 失敗ジョブの `詳細` 導線追加

### 中

- ダウンロード一覧の詳細ペイン追加
- フィルタ/並び替え追加
- 初回設定ガイド追加
- キューから設定タブへの保存先追加短絡導線

### 大

- ジョブカード化や複合ペイン化などの大幅な情報設計変更
- ダウンロード一覧と通知の統合的なインボックス化

## 今やるべきでない案

- 画面デザインだけを先に大きく変えること
- Electron 化や配布形態変更のような構造論へ飛ぶこと
- 失敗導線や状態可視化が弱いまま機能だけ増やすこと

## 採用優先順位

1. 失敗理由と復旧導線の強化
2. キュー上部の主操作と状態要約
3. BatchDialog の重複排除と件数追従
4. 通知タブの進行中フィードバック
5. フィルタ/並び替え
6. 詳細ペイン

## 未決事項

- 失敗行の復旧導線を `一覧内ボタン` にするか `詳細ペイン` に寄せるか
- フィルタを DB クエリに寄せるか、まずは UI 側絞り込みで始めるか
- 初回ガイドを常設表示にするか、初回のみ案内にするか
