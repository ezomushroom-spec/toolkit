# Project Rules

- この project は workspace の `AGENTS.md`、skills、docs、入口文書の棚卸しと再編判断を残すための監査用フォルダとして扱う。
- 実装コードの正本はこの project には置かず、判断記録と監査メモを正本として扱う。
- 読む先、文書更新先、判断記録先はこのフォルダ内に揃える。
- 参照対象は workspace root の `AGENTS.md`、`.agents/skills`、`.codex/`、`docs/`、`projects/` であり、必要な差分だけを記録する。
- 実装コード、依存設定、build 設定の変更は明示依頼がない限り行わない。
- 監査では「現状確認」「採用判断」「未決事項」を分けて残し、既存メモの重複再調査を避ける。

## build / run / test / lint

- 開発起動: なし
- 本番起動: なし
- テスト: 実ファイルと文書の突き合わせ
- lint / format: なし
- 実行に必要な前提: workspace 内の実ファイルを読めること

## 優先確認項目

- 正常系: 入口文書の読み順と正本境界が分かること
- 失敗系: 文書だけ読んでも誤った正本や保存先を指さないこと
- 空入力: 該当なし
- 不正設定: 該当なし
- 存在しないパス: 文書参照先の実在
- 途中停止やキャンセル: 調査途中でも判断済み事項を壊さないこと
- 危険操作: 既存 project の実装や設定を書き換えないこと
- メモリ / GPU / 長時間処理: 該当なし

## 変更時の必須セット作業

- 更新する docs: `AGENTS.md`、`START_HERE.md`、`PROJECT_SUMMARY.md`、この project 配下の監査メモ
- 最低限の確認コマンド: 参照した path や文書名が実在することを再確認する
- 最低限の手動確認: 初見で読んだときに、監査対象、正本文書、更新先が迷わず分かること
- 失敗時に先に見るログやファイル: root `AGENTS.md`、`docs/README.md`、この project 配下の最新監査メモ
