# 現状確認

## 案件概要

- 案件名: 生成画像向け仕上げレシピ適用ツール
- 種別: 新規アプリ提案
- 目的: 生成画像に対する定型的な仕上げ工程を、順序付きレシピとして保存し、単画像調整と複数画像への一括適用を安定して行える専用ツールを構築する
- 対象ユーザー: 生成画像を仕上げたい個人ユーザー

## 依頼の本質

- 主眼は汎用画像編集ではなく、生成画像の仕上げ品質を安定して再現することにある
- 中核価値は、補正機能の数ではなく、工程の組み合わせをレシピとして再利用できることにある
- 単画像プレビューと一括適用で、できるだけ同じ工程定義を使う必要がある

## 優先順位

1. 専門性
2. 機能的な GUI
3. 安定した動作

## スコープ

### 初期版で行うこと

- 単画像読込
- 仕上げ工程の調整
- 順序付きレシピの定義
- レシピ保存と再読込
- 補正前後比較
- 単画像への適用と保存
- 複数画像への一括適用
- 進捗表示
- 失敗一覧表示
- 日本語パスを含むローカルファイル対応

### 初期版で行わないこと

- 汎用ペイントソフト級の自由描画
- 本格的な万能画像編集
- 動画編集
- 3D 編集
- クラウド共同編集
- 高度なアセット管理
- 汎用レイヤーペイント
- 部分適用マスク
- 複数レシピ比較

## 中核設計方針

- 補正は単発フィルタではなく、順序付き工程列として扱う
- レシピは工程列の保存単位とする
- UI と処理本体を分離する
- プレビューと本適用は同じレシピ定義を共有する
- 元画像は上書きしない
- GPU は加速手段であり、初期版の前提条件にはしない

## 初期版の主要工程

- `color_smoothing`
- `blend_softening`
- `tone_smoothing`
- `brightness_adjust`
- `saturation_adjust`
