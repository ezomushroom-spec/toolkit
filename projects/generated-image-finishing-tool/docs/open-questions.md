## 再評価が必要になる条件

### GEGL 型工程モデル

以下の状況になった場合に再評価する。

- 直列工程だけでは仕上げ品質が足りない
- 部分適用や限定補正を入れる際に直列モデルが破綻し始める
- プレビュー高速化で中間結果再利用が重要になる
- 分岐的な処理表現が必要になる

### UI 基盤の再評価

以下の状況になった場合に再評価する。

- 比較表示や描画制御がデスクトップ GUI では不足する
- UI 表現上の要求が大きく変わる
- Web 技術側に移る合理的理由が明確になる

## ユーザー確認が必要な分岐

- 初期版から部分適用やマスクを入れる場合
- GPU を初期版必須へ引き上げる場合

- 汎用編集機能を追加して専用ツールの境界を広げる場合
