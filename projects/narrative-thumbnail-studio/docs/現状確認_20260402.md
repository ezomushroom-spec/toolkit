## 用途の整理

- 複数画像をプリセットに沿って並べる
- プレビュー上で分割比率やクロップ位置を調整する
- JPEG として書き出す
- 書き出し履歴を確認する

## 現在の技術構成

- Python
  - ローカル完結の画像処理と GUI 制御を 1 言語でまとめている
- PySide6
  - デスクトップ UI とイベント処理を担当
- Pillow
  - リサイズ、合成、文字描画、保存を担当
- NumPy / OpenCV
  - マスク生成、斜めワイプ、ラプラシアン計算などを担当
- mediapipe
  - `requirements.txt` と起動前ログ抑制設定には残るが、現行コードでは顔検出の実処理が外れている

## コード上の責務分担

- `ui/`
  - 画像プール、中央プレビュー、プリセット設定、履歴表示
- `core/layout_engine.py`
  - プリセットごとのレイアウト合成
- `core/exporter.py`
  - 書き出しと履歴管理
- `core/image_pool.py`
  - 画像読み込み、サムネイル、簡易キャッシュ
- `presets/*.json`
  - プリセット定義の読み込み元

## 技術的に重要な観点

- 通信やサーバーを持たないローカル単体ツールなので、Web フロント/バックエンド分離は必須ではない
- 画像合成処理は Python 側に既にまとまっており、別スタックへ移す利益は小さい
- UI は複雑すぎず、PySide6 のままで十分に扱える
- 依存の整理不足と、出力仕様の整合不足のほうが、フレームワーク選定より大きな問題になっている

## 現時点で確認した不整合

- EXIF 回転を見ていないため、画像の向きが崩れる可能性がある
- 単体出力と全プリセット出力で、使用画像やパラメータ反映の挙動が一致していない
- 履歴クリアが UI 上だけで、永続履歴には反映されない
