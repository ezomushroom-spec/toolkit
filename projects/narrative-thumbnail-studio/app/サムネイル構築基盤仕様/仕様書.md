# Narrative Thumbnail Studio 構築・仕様書

本ドキュメントは、Pixiv等のプラットフォームに向けたサムネイル画像を直感的な操作で構築・検証するためのデスクトップアプリケーション「Narrative Thumbnail Studio」を構成する技術・設計要素をまとめた仕様書です。

## 1. アプリケーション概要と設計思想

* **概要**:
  サムネイル画像のレイアウトプレビューやトリミング位置の最適化を行うための専用ツール。184×184pxのPixiv環境シミュレーションとフルサイズプレビューを切り替えながら確認できる。
* **設計思想 (UI/UX)**:
  「操作負担の最小化」「直感的UI」「柔軟性」を徹底。画像の追加や順序入れ替え、クロップ範囲の調整といった主要な操作のほとんどを、ドラッグ＆ドロップ（D&D）で直感的に完結させる構造としている。
* **パフォーマンスへの配慮**:
  重い画像合成処理やキャッシュ生成処理をバックグラウンドスレッドで非同期化し、ユーザーのドラッグ＆ドロップやUI操作に対するレスポンスをブロックしないようにしている。

## 2. 動作環境・技術スタック

* **動作OS**: Windows
* **フロントエンド**: Python + PySide6
* **画像処理**: Pillow (PIL)
* **起動方法**: `起動.bat` を介したダブルクリック起動（`pythonw` によるコンソール非常駐実行）に対応。日本語パスのフォルダでもパス解決が正しく行われるよう `pathlib.Path` で標準化済み。

## 3. ディレクトリ・アーキテクチャ構成

保守性および将来の機能拡張に備え、「ロジック層」と「UI層」を完全に分離したアーキテクチャを採用している。

### 3.1. `core/` (コアロジック層)
UIに依存しない画像処理やデータ管理に特化したビジネスロジックモジュール群。

* **`layout_engine.py`**: 画像の合成・トリミング・レイアウト処理を担う中核。非同期処理を前提とした設計。
* **`image_pool.py`**: 読み込まれた画像データをメモリ内で効率的に管理するプール機構。最大10画像までのLRU（Least Recently Used）キャッシュを搭載し、メモリの肥大化を防ぐ。
* **`presets.py`**: レイアウトのパターン定義（JSON）を読み込み、アプリケーション内で利用可能なデータ構造にパースする管理モジュール。
* **`text_overlay.py`**: サムネイル画像に対するテキスト描画等の合成エフェクトを担当。
* **`exporter.py`**: 検証・構築した最終的な合成結果を、ファイルとして出力・保存するエクスポート機能。

### 3.2. `ui/` (ユーザーインターフェース層)
PySide6を用いたウィジェット、レイアウト定義群。

* **`main_window.py`**: 全体を統括するメインハブとしてのウィンドウ。
* **`image_pool_panel.py`**: 左側パネル。D&Dでの画像追加、リスト上でのD&Dでの順番入れ替え操作を担当する。
* **`preset_panel.py`**: 右側上部パネル。現在利用可能なJSON設定プリセットの選択一覧UIを提供する。
* **`history_panel.py`**: 右側下部パネル。操作履歴を表示し、Undo (元に戻す) / Redo (やり直す) による試行錯誤をサポートする。
* **`preview_panel.py`**: 中央のメインパネル。リアルタイムの合成プレビューを描画する。画像上でのD&Dによってクロップ位置（オフセット）を動的に調整できる機能と、Pixiv用の縮小UIシミュレーション切替を備える。

## 4. 主なユーザビリティ配慮機能（実装済要件）

ユーザービリティを高める基本ルールに基づき、以下の機能が標準実装されている。

1. **D&Dによる完全な操作**: 外部からのファイルD&D追加はもちろん、プール内での順序入れ替え、プレビュー領域でのクロップ操作などの負担を最小限に抑えている。
2. **高速なリアルタイム描画**: LRUキャッシュおよび別スレッドでの処理（`FillCacheWorker` 等）により、スライダーやD&D変更時の画面のチラつきと遅延を抑制。
3. **日本語環境の完全サポート**: Windows特有のマルチバイトパス解決バグやファイルの文字化けに対し、Python標準の `pathlib` と `UTF-8` エンコーディングで徹底的に対策している。
4. **即座なやり直し**: 履歴管理を用いた Undo / Redo で失敗を恐れずにレイアウトを試せる。
