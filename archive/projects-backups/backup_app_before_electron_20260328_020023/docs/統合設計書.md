# 統合自動投稿ツール 設計書

**Project:** Multi-Platform Content Uploader  
**Date:** 2025-12-01

---

## 1. 概要

Pixiv、Patreon、Discordへのコンテンツ投稿を半自動化する統合ツール。
「制作完了」→「前処理」→「各プラットフォームへの投稿」というワークフローを採用し、手作業によるミス削減と効率化を目指す。

## 2. ワークフロー

### Phase 1: 前処理 (Pre-process)
1.  **Clean:** 元画像からメタデータ（Exif等）を除去し、投稿用画像を生成。
2.  **Zip:** 画像をアーカイブ化（Zip）する。
3.  **Upload:** ZipファイルをMEGAへアップロードし、共有URLを取得する。

### Phase 2: 投稿 (Publish)
*   **Pixiv:** 既存の自動化スクリプト（Browser Automation）を使用。
*   **Patreon:** ブラウザ自動化（Browser Automation）を使用し、画像とテキストを投稿。
*   **Discord:** Webhookを使用し、ZipファイルのURL（MEGA）とサンプル画像を投稿。

## 3. システム構成

```text
post_manager/
  ├─ data/
  │   ├─ posts.csv          # 統合管理ファイル
  │   └─ images/            # 画像フォルダ
  │
  ├─ dist/                  # Zip出力先
  ├─ config/
  │   └─ secrets.yaml       # アカウント情報 (Git除外)
  │
  └─ src/
      ├─ manager.py         # 統合実行くん (未実装)
      ├─ preprocessor.py    # 前処理 & MEGAアップロード (次フェーズで実装)
      ├─ uploader_pixiv.py  # Pixiv用 (既存 uploader.py)
      ├─ uploader_patreon.py# Patreon用 (新規)
      ├─ uploader_discord.py# Discord用 (新規)
      └─ verify_mega.py     # MEGA検証用
```

## 4. データ管理 (`posts.csv`)

| カラム名 | 説明 |
| :--- | :--- |
| `target_folder` | 画像フォルダパス |
| `title` | タイトル |
| `caption` | 説明文 |
| `tags` | タグ |
| `schedule` | 予約投稿日時 |
| `zip_url` | (自動入力) MEGAの共有URL |

## 5. 技術スタック

*   **Language:** Python 3.x
*   **Browser Automation:** Playwright (Pixiv, Patreon)
*   **Image Processing:** Pillow (Metadata removal)
*   **Cloud Storage:** mega.py (MEGA API)
*   **Notification:** Discord Webhook
